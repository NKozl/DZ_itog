# Итоговое задание

## Автоматизированный мониторинг и реагирование на угрозы


## Источники данных

### 1\. Логи Suricata

Используется файл `data/alerts-only.json`.

Что делает программа с логами:

* загружает JSON;
* превращает записи в таблицу pandas;
* фильтрует подозрительные события;
* считает статистику;
* показывает несколько примеров событий;
* строит график по source IP.

### 2\. API Vulners

Демонстрация работы с внешним API.

* ключ берётся из переменной окружения `VULNERS_API_KEY`;
* если ключ не задан, программа не падает, а выводит понятное сообщение.

## Логика проекта

### Фильтр для логов

Событие считается подозрительным, если `alert.severity` равно `1` или `2`.

### Реагирование

Если один и тот же **внешний** source IP встречается в подозрительных событиях 3 раза и больше, программа выводит сообщение об **имитации блокировки IP**.

## Установка зависимостей

Откройте терминал в папке проекта и выполните:

```bash
pip install -r requirements.txt
```

## Запуск проекта

1. ### Проверить часть с логами

```bash
python main.py
```

### 2\. Проверить и логи, и API Vulners

Сначала необходимо задать переменную окружения.

#### Windows PowerShell

```powershell
$env:VULNERS_API_KEY="ВАШ_КЛЮЧ"
python main.py
```

#### Windows cmd

```cmd
set VULNERS_API_KEY=ВАШ_КЛЮЧ
python main.py
```

#### Linux / macOS

```bash
export VULNERS_API_KEY="ВАШ_КЛЮЧ"
python main.py
```

## Что получится на выходе

После запуска создаются файлы:

* `outputs/report.json` — основной итоговый отчёт;
* `outputs/top_source_ip.png` — график топ-5 source IP по подозрительным событиям;
* `outputs/report_vulners.json` — сохранённые результаты запросов к Vulners API.

## Что находится в report.json

В отчёте сохраняется:

* общая информация о проекте;
* статистика по логам Suricata;
* правило фильтрации;
* примеры найденных событий;
* результат имитации реагирования;
* результат работы с Vulners API;
* пути к выходным файлам.

## Объяснение логики

Сначала программа читает логи Suricata из JSON-файла. Потом берёт из каждой записи самые важные поля: IP-адреса, порт, тип протокола, категорию и уровень опасности. После этого программа отбирает только более опасные события — те, у которых severity 1 или 2. Дальше она считает статистику и выбирает несколько примеров. Если какой-то внешний IP слишком часто появляется в подозрительных событиях, программа уведомляет, что такой IP нужно блокируется. Отдельно программа делает несколько запросов в Vulners API. Она получает JSON-ответ, берёт из него основные поля и сохраняет несколько примеров в отдельный файл и в общий отчёт.
